VAR-example

INPUT

var city; (DECLARING )
var city = (value); {ASSIGNING}

one input

var city = bangalore ;
console.log(city);

OUTPUT

bangalore

two inputs

var city = bangalore ;
var city = chennai ;
console.log(city);

OUTPUT

chennai

DEFINITION

  • we can re-declare and re-assign values
  • In the above example you can see i am having two variables with same name and value , the problem in var in it will take both into memory location eventhough it is having same values

LET-example

INPUT

let length; (DECLARING )
let length = (value); {ASSIGNING}

one input

let length = 10 ;
console.log(length);

OUTPUT

10

two input with different values

let length = 10 ;
length = 20;
console.log(length);

OUTPUT

20

two inputs with same let

let length = 10 ;
let length = 20 ;
console.log(length);

ERROR

Uncaught SyntaxError: Identifier 'length' has already been declared

DEFINITION

  • we cannot re-declare but we can re-assign values
  • In the above example you can see i am having two variables with same name and value , but let is showing error , this concept is mainly to avoid duplicate storage in memory location

CONST-example

INPUT

const pie; (DECLARING )
const pie = (value); {ASSIGNING}

one input

const pie = 3.141 ;
console.log(pie);

OUTPUT

3.141

two input with different values

const pie= 3.141 ;
pie = 2.9;
console.log(pie);

ERROR

Uncaught TypeError: Assignment to constant variable.

two inputs with same let

const pie = 3.141 ;
const pie = 20 ;
console.log(pie);

ERROR

Uncaught SyntaxError: Identifier 'pie' has already been declared

DEFINITION

  • we cannot re-declare and re-assign values
  • In the above example you can see i am having two variables with same name and value , but const is showing error , this concept is mainly to avoid duplicate storage in memory location and to avoid errors in inputs.
